Searching

You can use Python's regex syntax to search for anything in the logs. Note that this interface does not support plus signs (+), question marks (?) and ampersands (&). If you are unsure of your syntax, check it with this online tool.

Cheat Sheet

  • Everything:   .
  • Errors:   error
  • SSH Sessions for Root:   sshd.*root
  • Failed SSH Sessions:   sshd.*Failed password
  • HTTPD Erors:   httpd.*\[error\]
  • Entries for December 22nd Between 2-4 PM:   Dec 22 1[4-6]
  • Undeliverable Messages:   undeliverable address
  • Sudo Commands:   sudo
  • Microsoft Security Logs:   MSWinEventLog.{3}Security
  • Email Chains:   SUBJ.{0,4}((Fwd|FW).{0,4}){2,9}
  • Email with "Facebook" or "LinkedIn" in the Subject:   SUBJ.{0,15}(LinkedIn|Facebook)

Adding Logs

FLS looks in the default log directory (/var/log/remotesystems/) and adds every file found to it's "File" selector. Note that the user which Apache runs under must have read access to these files for FLS to search them!

Allow Apache To Read Logs

sudo chmod o+r -R /var/log/remotesystems/

Options

  • Update: Updates the results based on the current query every 10 seconds.
  • Top to Bottom: By default, FLS starts at the bottom of the log file(s) when searching. With this option enabled, it searches from the top to bottom.
